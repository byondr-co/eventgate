## Operational gotchas (cumulative learnings)

These are documented in [`docs/plans/improvement-and-findings-logs.md`](plans/improvement-and-findings-logs.md) — surface them to any new operator:

1. **Fly SSH does NOT inherit the Docker `ENV PATH`.** Bare `python manage.py X` inside `flyctl ssh console` (interactive or `--command`) fails with `ModuleNotFoundError: No module named 'django'` because the venv path isn't in the bash shell's PATH. Use `/app/.venv/bin/python manage.py …` explicitly. `uv` is also not in the runtime container's PATH.
2. **`flyctl secrets set` does NOT run `release_command`.** For env vars that need a Django management command to take effect (e.g., `TELEGRAM_WEBHOOK_URL` requires `setup_telegram_webhook`), run the command manually after `flyctl secrets set`. Or push any commit to trigger a full GHA deploy.
3. **mypy local (`apps/`) vs CI (`apps config`) scope mismatch.** Settings-level type bugs (e.g., `MEDIA_ROOT` type) slip past local pre-commit and only fail in CI. **TODO:** normalize both to the same scope.
4. **Telegram BotFather doesn't allow bot username changes.** Only display name + other attributes are editable. To rename, create a new bot and migrate. Old bot can be left as a safety net.
5. **Vercel `NEXT_PUBLIC_*` env vars inline at build time** (including in Server Components). After setting/changing a `NEXT_PUBLIC_*` var, trigger a redeploy.
6. **gh CLI active account can flip between authenticated accounts.** This repo is on `vineidev`; if `gh pr create` errors with "Could not resolve to a Repository," check `gh auth status` and run `gh auth switch --hostname github.com --user vineidev`.

### Hygiene wave — 3 bugs surfaced during Plan H T9 smoke

All pre-existing (not Plan H regressions), all visible in [`docs/plans/improvement-and-findings-logs.md`](plans/improvement-and-findings-logs.md):

1. **Device creation: 500 + raw HTML on unique (Label, Role) violation.** `IntegrityError` bubbles up unhandled. Fix recipe in log: wrap save in try/except `IntegrityError` and re-raise as `ValidationError` in the serializer, or add an explicit serializer-level uniqueness validator.
2. **Event-list status badge fixed to "draft".** UI badge isn't bound to row's `status` field; always renders the literal string. Per brief §5 schema, the field can be `'draft' | 'open' | 'live' | 'closed' | 'archived'`.
3. **UI lacks structural navigation.** No breadcrumbs, no consistent back button, no hierarchy indicator. Operators rely on browser back / URL surgery. Recommend shadcn Breadcrumb + persistent sidebar or tab nav for the event-context routes.

Also worth adding to the wave:
